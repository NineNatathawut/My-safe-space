# 👥 บ้านพักใจ (My Safe Space) - Roles & Permissions System

ไฟล์นี้สรุปโครงสร้างสิทธิ์ (Roles), บทบาทหน้าที่ (Permissions), และแนวทางการเข้าถึงระบบของผู้ดูแลระบบ (Admin) เพื่อให้ AI Developer ใช้เป็นแนวทางในการกำหนด Logic ความปลอดภัยและการเข้าถึงข้อมูล (Access Control)

---

## 🔍 เจาะลึกกลไกของแต่ละบทบาท (Role Specifications)

### 1. Guest (แขกผู้มาเยือน)
- **การพิสูจน์ตัวตน:** ไม่ต้องล็อกอิน ระบบจำผ่าน `localStorage` (ใช้ `guest_token`)
- **ชื่อที่แสดง:** ระบบสุ่มให้ใหม่ทุกครั้งที่กดสร้างโพสต์ (เช่น `🌙 ดาวเหนือ`)
- **ข้อจำกัด:** โพสต์ระบายได้ 1 ครั้ง และตอบกลับได้ 1 ครั้ง ข้อมูลประวัติจะคงอยู่ตราบเท่าที่ไม่ได้ล้างแคชของเบราว์เซอร์

### 2. Member (สมาชิกบ้านพักใจ)
- **การพิสูจน์ตัวตน:** ล็อกอินผ่านระบบ Auth (เช่น อีเมล/รหัสผ่าน หรือโซเชียล) แต่**ไม่แสดงข้อมูลจริงบนหน้าเว็บ**
- **ชื่อที่แสดง:** ระบบจะ "ล็อกชื่อสุ่ม" ที่เขาชื่นชอบไว้เป็นชื่อประจำบัญชีถาวร
- **จุดเด่น:** สามารถดูประวัติบันทึกใจของตัวเองย้อนหลังได้จากทุกอุปกรณ์ และเข้าถึงระบบแจ้งเตือนแบบเรียลไทม์

### 3. Listener (ผู้ฟัง / อาสาสมัครเยือนใจ)
- **การพิสูจน์ตัวตน:** สมัครสมาชิกตามปกติ แต่ต้องได้รับการกดยืนยันสิทธิ์ (Approve) จาก Admin หลังบ้าน
- **เป้าหมาย:** กลุ่มคนที่มีพลังบวก หรือนักจิตวิทยาที่ต้องการเข้ามาช่วยตอบคอมเมนต์เยียวยาผู้ใช้งาน
- **ชื่อที่แสดง:** มีคำนำหน้าเพื่อสร้างความน่าเชื่อถือ เช่น `🌸 อาสาสมัคร เมฆสีฟ้า`

### 4. Admin / Mod (ผู้ดูแลบ้านพักใจ)
- **การพิสูจน์ตัวตน:** บัญชีสิทธิ์สูงสุด ผูกกับอีเมลของผู้พัฒนา (ตัวคุณเอง) และทีมงานโดยตรง
- **ชื่อที่แสดง:** แสดงสถานะผู้ดูแลชัดเจนเมื่อทำหน้าที่ควบคุมระบบ
- **หน้าที่หลัก:** ทำหน้าที่เป็น "คนกวาดบ้าน" คอยลบโพสต์ที่เป็นภัย (Toxic/Troll) และจัดการเนื้อหาบทความสุขภาพจิต

## 🔑 กลไกการเข้าสู่ระบบของ Admin (Admin Authentication Flow)

เพื่อความปลอดภัยและความเป็นส่วนตัวสูงสุดของระบบ ระบบแอดมินจะถูกแยกออกจากส่วนของผู้ใช้งานทั่วไปดังนี้:

### 1. ทางเข้าลับ (Hidden Portal)
- จะไม่มีปุ่ม "เข้าสู่ระบบแอดมิน" ปรากฏบนหน้าแรก (Home Page)
- แอดมินต้องเข้าใช้งานผ่าน URL เส้นทางเฉพาะ (Hidden Route) เท่านั้น เช่น `/backoffice` หรือ `/admin-login`

### 2. การกำหนดสิทธิ์สถาปนิก (Database Seeding)
- ในขั้นตอนการตั้งระบบฐานข้อมูลครั้งแรก (Initial Setup) อีเมลของคุณจะถูกระบุสถานะเป็น `role: 'ADMIN'` ในฐานข้อมูลโดยตรงผ่านโค้ดหลังบ้าน (Seed Script)
- เมื่อคุณล็อกอินผ่านหน้าทางเข้าลับ ระบบ Next.js 15 จะเช็กสิทธิ์จากฐานข้อมูล หากตรงกันจะทำการเปิดหน้า **Admin Dashboard** ให้ทันที

### 3. ห้องควบคุมหลังบ้าน (Admin Dashboard Features)
- **Content Moderation:** หน้าต่างมอนิเตอร์โพสต์ล่าสุด พร้อมปุ่ม [ลบโพสต์] หรือ [เตือนผู้ใช้]
- **User Management:** รายชื่อบัญชีผู้ใช้ (แสดงเฉพาะชื่อสุ่ม) พร้อมปุ่ม [แบนชั่วคราว] หรือ [แต่งตั้งเป็น Listener]
- **Resource Management:** เครื่องมือแก้ไข/เพิ่ม ลิงก์ยูทูป บทความ และข้อมูลสายด่วนสุขภาพจิต
